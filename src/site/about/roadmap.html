TritonAI Roadmap

Where TritonAI Is Going Next

The next phase expands beyond chat into a broader tools hub: richer model access for developers, more embedded campus experiences, workflow orchestration, observability, live data integrations, and new support for academic and administrative use cases.

In Progress

  • TritonAI Harness Development — Campus agentic development platform. V1 Mac feature set complete June 2026. Agent collaboration/social layer for distributed coordination with governance.
  • Instructional AI Expansion — Shifted from local OSS models to cloud frontier models (Claude Sonnet, Gemini 3.5 Flash) to remove model quality as a variable during evaluation. More course pilots and faculty tooling.
  • Developer API Expansion — Self-hosted model credits, approved cloud routes, usage reporting, and clearer paths from prototype to funded campus application.
  • Student Scheduling Assistant — Launched July 9, 2026, serving 20K-40K students in waves.
  • Mobile Integration — UC San Diego Assistant now live in the campus mobile app.
  • Model & Service Observability — Catalog monitoring, access checks, usage signals, and AI-assisted detection for service anomalies.

Planned

  • Transcript Verification — OCR and transcript matching workflows for enrollment-related review.
  • Academic Advising Agent — Proof-of-concept for tier-one advising questions and triage.
  • Agent Builder & Workflow Tools — Low-code orchestration, reusable skills, MCP-style connectors, and guardrails for supervised multi-step campus workflows.
  • UC-Wide Claude Enterprise Agreement — UCOP working with Anthropic on system-wide agreement.
