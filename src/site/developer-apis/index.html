TritonAI Harness: Campus Agentic Development Platform

The TritonAI Harness is UC San Diego's campus agentic development platform — the execution layer that turns model inference into governed, tool-using AI workers. Based on the Hugging Face agent glossary: Model ≠ Agent. Agent = Model + Scaffold + Harness. The Harness is what makes it one: it calls the model, routes tool calls, manages the action-observation loop, handles errors, and controls state and memory across turns.

Architecture: User → Harness → LiteLLM Gateway → Hyperscaler/On-prem models + Tools/Skills (via MCP)

The Harness evolved from the T3 Code fork and was formally named "TritonAI Harness" in June 2026. V1 (Mac) feature set is complete, including cron scheduling, heartbeat monitoring, job execution, and MS Graph access. It functions as UCSD's branded Codex replacement with governed gateway metering, observability, and recharge mechanics.

Key capabilities include:

  • Agent orchestration — multi-step workflow execution with tool calls, error handling, and state management
  • MCP-based connectors — reusable skills and tools from the UCSD Skills Library, including MS Graph, Confluence, and campus data integrations
  • Governed access — every request routes through the LiteLLM gateway with usage tracking, access scopes, and model routing policies
  • Citizen developer pathway — tiered deployment model from individual experimentation to campus-wide enterprise applications
  • Recharge mechanics — usage-based cost recovery for production projects that scale beyond starter credits

Developer API Architecture

Campus users connect through AI Harnesses (Claude Code, Codex, TritonBuild) to the LLM Gateway, which routes to both on-prem models at the San Diego Supercomputer Center and approved cloud providers. The gateway enforces templates, guardrails, and usage tracking. ITS owns templates, hosting, and baseline risk review; departments own app logic, accessibility, testing, and end-user support.

Tiered Deployment Model

From individual citizen development to campus-wide enterprise deployment — a structured escalation path with review gates at each tier.

What You Get

  • API key for the LiteLLM gateway at tritonai-api.ucsd.edu/v1
  • Default credits ($15/month) for experimentation
  • On-prem models — self-hosted open-weight models inside the UC firewall at SDSC, including Llama, Mistral, Gemma, and DeepSeek variants, with no per-token cost for campus users
  • Cloud models — access to 20+ approved cloud models including GPT-5.5, Claude Opus 4.7, Claude Sonnet 4.6, Gemini 3.1 Pro/Flash, Amazon Nova Premier, and Cohere transcription
  • Usage reporting and recharge mechanics for production projects
  • MCP-based connectors and reusable skills from the UCSD Skills Library

On-Prem Model Hosting at SDSC

The San Diego Supercomputer Center hosts open-weight models inside the UC firewall, giving the campus a resilient, cost-controlled AI infrastructure that doesn't depend on external API availability or vendor rate limits. This is central to UC San Diego's strategy: own the base layer, rent the frontier.

  • Data sovereignty — sensitive campus data never leaves the UC network when using on-prem models
  • Cost control — no per-token charges for self-hosted models; cloud usage is opt-in for frontier capabilities
  • Vendor independence — reduce lock-in to any single cloud provider; on-prem models provide a fallback if cloud access changes
  • Research access — SDSC GPU infrastructure (including DGX-class hardware) available for research and instructional workloads
  • Model flexibility — open-weight models can be fine-tuned, quantized, or swapped without vendor approval

Cloud Model Access

For workloads that need frontier capabilities, the LiteLLM gateway provides governed access to approved cloud providers with the same usage tracking, access scopes, and recharge mechanics as on-prem models. Cloud and on-prem models can be mixed within a single agent workflow — the Harness routes to whichever model the gateway policy specifies.

Cloud (via gateway): GPT-5.5, Claude Opus 4.7, Claude Sonnet 4.6, Gemini 3.1 Pro/Flash, Amazon Nova Premier, Cohere transcription.

  • UCSD-Skills-Library (public) — all other skills in tritonai/ and community/ folders

Agent Engineering Strategy

TritonAI supports multiple agentic coding interfaces so builders can choose the right tool for the task. All routes connect through the same LiteLLM gateway with campus recharge and on-prem governance.

  • CODEX (OpenAI) — autonomous delegation for self-contained, high-correctness tasks. Best when accuracy and verification are non-negotiable and work can run in parallel isolated environments.
  • Claude Code (Anthropic) — collaborative coordination across GitHub, docs, chat, and planning tools. Best for interdependent work where teams and agents must stay aligned with human-in-the-loop iteration.
  • Gemini Antigravity (Google) — also supported for teams that prefer the Gemini ecosystem.

Routing guidance: Use CODEX when correctness and autonomous recovery are critical and the task is self-contained. Use Claude Code when iterative drafting and review are acceptable and the workflow spans multiple systems.

Campus App Hosting

What happens after you build — the hosting path for campus apps, with review gates at each step.

StepPhaseWhat happens
1PrototypeBuild locally with Claude Code, CODEX, or curated ITS templates through the Citizen Developer Program
2ApplySubmit department sponsor, technical POC, audience, and data classification
3ReviewAutomated checks plus guided ITS review of data sensitivity, restricted APIs, and stack fit
4DeployCampus-only hosting with SSO, logging, and network controls — available on campus or VPN
5OperateDepartment supports the app; ITS manages platform health and may remove non-compliant apps
6PromoteHigh-value apps graduate to enterprise support; unsupported apps are retired
